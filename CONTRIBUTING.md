# Contributing

Feedback on the Protocol Field working paper is welcome. This includes typos, broken links, missing citations, unclear wording, or broader conversation about the framework.

You do not need prior GitHub experience.

You do not need to use the Git command line.

A free GitHub account is enough.

## Three ways to contribute

- **[Open an issue](https://github.com/johnlgrant-research/protocol-field/issues)** — errors, questions, or anything that should be tracked until resolved. An issue can point to a specific line of the paper. See [Commenting on a specific line of the paper](#commenting-on-a-specific-line-of-the-paper).
- **[Start a discussion](https://github.com/johnlgrant-research/protocol-field/discussions)** — open-ended exchange about a passage, the framework, related work, or applications. A discussion can also point to a specific line. See [Commenting on a specific line of the paper](#commenting-on-a-specific-line-of-the-paper).
- **Open a pull request** — propose a specific edit to [`protocol-field.md`](protocol-field.md), like Suggesting mode in Google Docs. See [Proposing an edit in the browser](#proposing-an-edit-in-the-browser).

## Commenting on a specific line of the paper

[`protocol-field.md`](protocol-field.md) displays as a formatted document by default, so line numbers and GitHub's line-selection menu are hidden.

To refer to a specific line in an issue or discussion:

1. Open [`protocol-field.md`](protocol-field.md) in this GitHub repository.
2. Near the top right of the file, click the **Code** button (between **Preview** and **Blame**). The file switches to its source view, with line numbers down the left.
3. Click the line number you want to reference. To select a range, click the first line number, then hold **Shift** and click the last.
4. A small line-selection menu appears next to your selection. Choose either:
   - **Reference in new issue** — opens a new issue form pre-filled with a quote of the lines you selected.
   - **Reference in new discussion** — opens a new discussion form pre-filled with a quote of the lines you selected.
5. Write your comment around the pre-filled quote, then submit.

GitHub documents the issue path in detail; the discussion path uses the same line-selection menu: [Creating an issue from code](https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/creating-an-issue#creating-an-issue-from-code).

If any of this is awkward, just open a plain issue or discussion and quote the passage by hand. A short excerpt and a section heading are enough for me to find it.

## Proposing an edit in the browser

A pull request is close to Suggesting mode in Google Docs. You propose a change, but it is not added to the main document until it has been reviewed and accepted.

1. Open [`protocol-field.md`](protocol-field.md) in this GitHub repository.
2. Click the pencil icon at the top right of the file. If you cannot edit the file directly, GitHub will make a copy under your account so you can propose the change.
3. Edit the text. Use the **Preview** tab to check formatting.
4. At the bottom, describe your change and click **Propose changes**.
5. On the next screen, click **Create pull request**.

GitHub handles the technical steps for you. Their step-by-step guide with screenshots is [Editing files in GitHub](https://docs.github.com/en/repositories/working-with-files/managing-files/editing-files).

## What makes a good pull request

- **Small and focused.** One typo, one rewording, or one citation per pull request. This makes it easier and faster to review.
- **A sentence of context** in the pull request description is enough. For larger rewordings, say briefly why.
- **For substantive changes** such as new sections, reframing an argument, or changes to the axes, please open an issue first so we can discuss before you invest editing time.

## About the figures

The 7 figures in [`figures/`](figures/) are generated from a separate source.

If you spot a factual problem with a figure, please open an issue rather than editing or replacing the PNG. Direct changes to the PNG may be lost when the figures are regenerated.
