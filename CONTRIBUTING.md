# Contributing

Feedback on the Protocol Field working paper is welcome — typos, citations, clarifications, or broader conversation about the framework. You do not need to use the Git command line.

## Three ways to contribute

- **[Open an issue](https://github.com/johnlgrant-research/protocol-field/issues)** — errors, questions, or anything you want tracked to resolution. From a line in [`protocol-field.md`](protocol-field.md), GitHub's line menu offers *Reference in new issue* to anchor the issue to that line.
- **[Start a discussion](https://github.com/johnlgrant-research/protocol-field/discussions)** — open-ended exchange about a passage, the framework, related work, or applications. *Reference in new discussion* works the same way from the line menu.
- **Open a pull request** — a specific proposed edit to [`protocol-field.md`](protocol-field.md). See the web-UI walkthrough below.

## Proposing an edit in the browser

1. Open [`protocol-field.md`](protocol-field.md) in the repository.
2. Click the pencil icon at the top right of the file. GitHub will fork the repo into your account automatically if needed.
3. Edit the text. Use the **Preview** tab to check formatting.
4. At the bottom, describe your change and click **Propose changes**.
5. On the next screen, click **Create pull request**.

GitHub handles the fork, branch, commit, and PR plumbing for you. Their step-by-step guide with screenshots is [Editing files in GitHub](https://docs.github.com/en/repositories/working-with-files/managing-files/editing-files).

## What makes a good pull request

- **Small and focused.** One typo, one rewording, or one citation per PR — easier and faster to review.
- **A sentence of context** in the PR description is enough. For larger rewordings, say briefly why.
- **For substantive changes** (new sections, reframing an argument, changes to the axes), please open an issue first so we can discuss before you invest editing time.

## About the figures

The 7 figures in [`figures/`](figures/) are generated from a separate source and are not editable as markdown. If you spot a factual problem with a figure, please open an issue rather than replacing the PNG.
